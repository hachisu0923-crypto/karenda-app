# 引き継ぎ（ローカル開発ガイド）— karenda / My Calendar

このリポジトリは **Web カレンダーアプリ（`karenda-/`）** と、それを **iPhone 風の Windows デスクトップアプリ**として包む **Electron ラッパー（`desktop/`）** の2層構成です。
ローカル（自分の PC）で開発・確認するための手順と、設計上の注意点をまとめます。

---

## 1. 構成（ざっくり）

```
karenda-app/
├── karenda-/                  # Web アプリ本体（バニラ HTML/CSS/JS + Supabase の PWA。ビルド不要）
│   ├── index.html            # 画面の HTML（モーダル・下部バー等もここ）
│   ├── app.js                # ロジック全部（カレンダー/予定/シフト/家計簿/タイマー/BGM 等）
│   ├── style.css             # スタイル（モバイル UI は @media (max-width:720px)）
│   ├── sw.js / manifest.json # PWA（Service Worker / マニフェスト）
│   ├── mobile-preview.html   # スマホ枠プレビュー（参考）
│   ├── supabase/             # Edge Functions / SQL マイグレーション
│   └── SETUP_*.md            # 通知・JCB 取込などの個別セットアップ手順
│
├── desktop/                   # Electron ラッパー（Windows .exe を作る）
│   ├── main.js               # メインプロセス（ウィンドウ生成・IPC・各種注入）
│   ├── preload.js            # レンダラへ最小ブリッジ（electronOAuth / electronControls）
│   ├── static-server.js      # 依存なしのローカル HTTP サーバ（karenda- を配信）
│   ├── shell.html            # iPhone 枠（黒ボディ＋ノッチ＋画面 iframe）。トップフレーム
│   ├── scripts/make-icon.js  # icon-192.png → build/icon.ico 生成
│   ├── package.json          # electron / electron-builder 設定
│   └── README.md
│
└── .github/workflows/build-windows.yml  # CI（windows ランナーで .exe 自動ビルド）
```

- **Web 本体は `karenda-/`**。ビルド工程は無く、静的ファイルをそのまま配信するだけ。
- **デスクトップ版は `desktop/`** が `karenda-/` をローカル配信して iframe で読み込む薄いラッパー。
- データは **Supabase**（`app.js` 内に URL と anon キー）。Supabase SDK は jsdelivr CDN から読み込む → **動作には常時インターネットが必要**。

## 2. ローカルでの起動

> 前提：Node.js（CI は **Node 20**）。`git` 。インターネット接続（Supabase / CDN / BGM の YouTube）。

### A. Web 版だけ手早く確認する
`karenda-/` を**HTTP で配信**して開く（`file://` は localStorage/SW の origin 問題で避ける）。

```bash
cd karenda-
npx serve .            # もしくは: python3 -m http.server 5500
# 表示された http://localhost:xxxx を開く
```

- ブラウザ幅を **720px 未満**にするとモバイル UI（デスクトップ版と同じ見た目）になります。
- ログイン（メール/パスワード or Google）も Web ではそのまま動きます。

### B. デスクトップ版（Electron）を起動する ← 普段の開発はこちら推奨

```bash
cd desktop
npm install            # 初回のみ（electron などを取得）
npm start              # = electron .  → iPhone 枠アプリが起動
```

- 起動すると `static-server.js` が **固定ポート 8923（埋まっていたら 8924→8925）** で `../karenda-` を配信し、`http://127.0.0.1:8923/__shell` を読み込みます。
- `karenda-/` を編集して**アプリを再起動**すれば反映されます（ホットリロードは無し）。
- DevTools を開きたいときは `main.js` の `createWindow()` 内に一時的に `mainWindow.webContents.openDevTools({mode:'detach'})` を足すのが手軽です（コミットはしないこと）。

## 3. Windows .exe のビルド

- **Windows でのみビルド可能**（electron-builder の `--win`）。Mac/Linux では `.exe` は作れません。

```bash
cd desktop
npm run dist     # NSIS インストーラ + portable を desktop/dist/ に生成
npm run pack     # 署名なしの展開フォルダだけ作る（動作確認用に速い）
```

- **Windows PC が無い場合は GitHub Actions に任せる**：
  - `.github/workflows/build-windows.yml` が `main` と `claude/stoic-euler-o43jvu` への push、`workflow_dispatch`、`v*` タグで起動。
  - 成果物はアーティファクト名 **`my-calendar-windows`**（`desktop/dist/*.exe`）。Actions の実行ページからダウンロード。
  - `v*` タグを打つと GitHub Release にも添付されます。

## 4. Supabase（バックエンド）

- 接続情報は **`karenda-/app.js` 冒頭**（`SUPABASE_URL` / `SUPABASE_KEY`(anon)）。プロジェクト ref は `oungvayvmxkszsokxwxd`。
- **Google ログイン（デスクトップ版）を使うには、Supabase ダッシュボードで手動設定が必要**：
  - Authentication → URL Configuration → **Redirect URLs** に以下を追加：
    - `http://127.0.0.1:8923/oauth-callback`
    - `http://127.0.0.1:8924/oauth-callback`
    - `http://127.0.0.1:8925/oauth-callback`
  - Authentication → Providers → **Google を有効**（Web 版で Google ボタンが出ていれば有効済み）。
  - Google Cloud 側の変更は不要（Google のリダイレクト先は Supabase の HTTPS コールバックのまま）。
- 通知・JCB 取込など追加機能のセットアップは `karenda-/SETUP_NOTIFICATIONS.md` / `SETUP_JCB_INGEST.md` を参照。

## 5. ★設計の要点・ハマりどころ（最重要）

ここを外すと事故ります。

1. **`karenda-/` は Web 版と共用。Web の挙動を壊さないこと。**
   デスクトップ専用処理は必ず `if (window.electronOAuth)` / `if (window.electronControls)` 等で**ガード**する（Web ではこれらは `undefined`）。

2. **ネイティブ `confirm()` / `alert()` / `prompt()` を使わない。**
   frameless + transparent の Electron ウィンドウでは、ネイティブダイアログ表示後に**ウィンドウが入力を受け付けなくなる**（要再起動）不具合が出る。
   → 代わりに **`appConfirm(message, okLabel)`（Promise<boolean>）** と **`appNotice(message)`（トースト）** を使う（`app.js` 内）。`window.alert` は既にトーストへ置換済み。

3. **デスクトップ版はカレンダーを iframe 内で動かす。** 論理幅は常に 390px ＝ 常にモバイル UI（`@media (max-width:720px)`）。
   - トップフレーム＝ `shell.html`（iPhone 枠）、画面部分＝ `<iframe src="/">`（= `karenda-/index.html`）。
   - `preload.js` は `nodeIntegrationInSubFrames:true` で **iframe にも注入**される。だから iframe 内 `app.js` から `window.electronOAuth` が使える。

4. **マウス操作はタッチに変換している。** `main.js` の `SYNTH_SRC` を iframe のメインワールドへ注入し、マウスのドラッグを `touchstart/move/end` に変換（指スワイプと同じ操作を実現）。
   - 予定のドラッグ＆ドロップ（`.event-pill[draggable]`）と入力欄/編集要素は**変換対象外**。
   - ドラッグ中のテキスト選択は `selectstart` 抑止＋`removeAllRanges` で能動的に止めている。

5. **配信は固定ポート 8923。** origin が安定するので localStorage（＝ログインセッション）が再起動後も維持される。OAuth のリダイレクト URL もこのポート前提。

6. **electron-builder の同梱対象。** `desktop/package.json` の `build.files` に `main.js / preload.js / static-server.js / shell.html / scripts/**` を必ず含める（`preload.js` 等を入れ忘れるとパッケージ版で黙って読み込まれない）。`karenda-/` は `extraResources` で `app-web` として同梱。

7. **透過ウィンドウの制約。** `transparent:true`＋`frame:false`。影は CSS（透明マージン内）。リサイズはアスペクト比固定（`setAspectRatio`）。最前面固定トグルは `shell.html` のピンボタン（状態は localStorage `karPinTop`）。

8. **このクラウド環境（Linux）では `.exe` ビルドも GUI 実行もできない。** 見た目・実挙動の最終確認は必ず Windows 実機（`.exe` か `npm start`）で行う。Supabase/CDN への到達も環境次第（egress 制限あり）。

## 6. これまでに入れた主な機能（参考）

- Google ログイン（PKCE + 規定ブラウザ + ループバック `/oauth-callback`）
- iPhone 風ウィンドウ（frameless/transparent、ノッチ、独自の最小化/閉じる、固定アスペクトのリサイズ、最前面固定トグル、右上ステータス時計）
- マウス→指スワイプ変換 / スワイプ時のテキスト選択抑止
- 月セルの予定時刻を**アナログ時計アイコン**で表示（午前=オレンジ / 午後・夜=青、日付番号の右に時間順）
- 下部バーに**タイマー**（カウントダウン＋ポモドーロ、通知＋ビープ）
- サイドバーに **YouTube BGM** 再生ボタン（ミニカレンダー下）
- 予定削除後にネイティブダイアログでウィンドウが固まる不具合の修正（→ アプリ内ダイアログ化）

## 7. Git / ブランチ運用

- 開発ブランチ：**`claude/stoic-euler-o43jvu`**（push すると CI が走る）。`main` が統合先。
- 変更 → push → CI（build-windows）成功を確認 → PR を作成して `main` へマージ、の流れ。
- コミット/PR には内部のモデル識別子等を含めない。
